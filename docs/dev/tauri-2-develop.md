<p>iOS</p>
<p>Safari must be used to access the Web Inspector for your iOS application.</p>
<p>Open the Safari on your Mac machine, choose <strong>Safari > Settings</strong> in the menu bar, click <strong>Advanced</strong>, then select <strong>Show features for web developers</strong>.</p>

<p>After following all steps you should see a <strong>Develop</strong> menu in Safari, where you will find the connected devices and applications to inspect.
Select your device or simulator and click on <strong>localhost</strong> to open the Safari Developer Tools window.</p>
</li>
<li>
<p>Android</p>
<p>The inspector is enabled by default for Android emulators, but you must enable it for physical devices.

This will enable Developer Mode for your Android device and the <strong>Developer Options</strong> settings.</p>
<p>To enable application debugging on your device you must enter the <strong>Developer Options</strong> settings, toggle on the developer options switch
and enable <strong>USB Debugging</strong>.</p>

<ol>
<li>Error running build script on Xcode</li>
</ol>
<p>Tauri hooks into the iOS Xcode project by creating a build phase that executes the Tauri CLI to compile the Rust source
as a library that is loaded at runtime. The build phase is executed on the Xcode process context, so it might not be able
to use shell modifications such as PATH additions, so be careful when using tools such as Node.js version managers which may not be compatible.</p>
<ol start="2">
<li>Network permission prompt on first iOS app execution</li>
</ol>
<p>On the first time you execute <code dir="auto">tauri ios dev</code> you might see iOS prompting you for permission to find and connect
to devices on your local network. This permission is required because to access your development server from an iOS device,
we must expose it in the local network. To run your app in your device you must click Allow and restart your application.</p>

<p>Similarly to how your webview reflects changes in real time,
Tauri watches your Rust files for changes so when you modify any of them your application is automatically rebuilt and restarted.</p>
<p>You can disable this behavior by using the <code dir="auto">--no-watch</code> flag on the <code dir="auto">tauri dev</code> command.</p>
<p>To restrict the files that are watched for changes you can create a <code dir="auto">.taurignore</code> file in the src-tauri folder.
This file works just like a regular Git ignore file, so you can ignore any folder or file:</p>
