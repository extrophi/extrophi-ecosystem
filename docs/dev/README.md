## Purpose

This directory contains:
- Architecture patterns and design decisions
- API documentation and specifications
- Database schemas and migrations
- Development guides and tutorials
- Code conventions and style guides
- Build and deployment procedures

## Adding Documentation

When creating new development documentation:

1. **Use clear, descriptive filenames** - e.g., `whisper-ffi-integration.md`, `svelte-5-migration-guide.md`
2. **Include code examples** - Show practical implementations
3. **Document "why" not just "what"** - Explain design decisions
4. **Keep it current** - Update docs when code changes
5. **Link related docs** - Reference other relevant documentation
